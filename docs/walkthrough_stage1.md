# 架构优化阶段一：主进程解耦与职责划分 重构总结

我们已经成功实施了优化计划的第一阶段，对主进程的 `SSHManager` 进行了彻底的解耦重构。

---

## 变更概述

这次重构将原先 640 余行的 `sshManager.ts` 拆解为职责明确的三个子模块，大大提高了代码的内聚性和可维护性：

### 1. `src/main/services/telemetryService.ts`
* 承接了远程主机系统信息嗅探和每 2 秒定期执行的 `/proc/` 数据拉取任务。
* 独立存放了所有纯文本解析逻辑（如 CPU、Memory、Network、Disk 和 Processes 的提取算法），使之易于后续扩展及编写单元测试。

### 2. `src/main/services/sftpService.ts`
* 承接了底层 SFTP 包装器的生命周期维护（`getSftp`）。
* 封装了目录操作（获取文件列表、新建文件夹、重命名、权限修改、删除）和传输进程（文件/文件夹下载与递归上传，并向渲染进程发送 `sftp:progress` IPC 进度更新）。
* 包含了通用的文件日志追加工具 `logToFile`。

### 3. `src/main/sshManager.ts`
* 瘦身为 170 行的轻量级会话池。仅保留 `sessions = new Map<string, Session>()` 和终端 Shell 字符流收发、终端大小调整等核心通道操作。
* 相应的 SFTP 业务操作与监控生命周期均通过调用上述两个 Service 的对应方法完成。
* 重新导出了 `logToFile` 以保持对 `index.ts` 接口的后向兼容性。
